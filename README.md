# Mailer

Единый веб-клиент для работы с несколькими почтовыми ящиками через IMAP и SMTP. Письма не хранятся в базе — загружаются по запросу с сервера.

## Возможности

- **Несколько ящиков** — Yandex, Mail.ru, Gmail, Outlook или произвольный IMAP/SMTP
- **Папки** — входящие, отправленные, архив, корзина, спам
- **Метки** — пользовательские метки с привязкой к письмам
- **Контакты** — сохранение адресов из писем
- **Поиск** — по всем ящикам и папкам (кроме спама), с бейджами папок в результатах
- **Составление писем** — ответ, вложения, подпись, отображаемое имя отправителя
- **Настройки** — управление ящиками: имя, цвет, подпись, имя отправителя, удаление

## Требования

- Node.js 18+
- npm

## Установка и запуск

```bash
# Клонировать репозиторий и перейти в каталог
cd Mailer

# Установить зависимости
npm install

# Настроить переменные окружения
cp .env.example .env.local
# Сгенерировать ключ: openssl rand -hex 32
# Вставить его в ENCRYPTION_KEY в .env.local

# Запуск в режиме разработки
npm run dev
```

Приложение откроется по адресу [http://localhost:3000](http://localhost:3000).

## Переменные окружения

| Переменная        | Описание |
|-------------------|----------|
| `ENCRYPTION_KEY`  | Ключ шифрования паролей ящиков (минимум 32 символа). Сгенерировать: `openssl rand -hex 32` |

## Добавление почтового ящика

1. Откройте **Настройки** в боковой панели.
2. На вкладке **Почтовые ящики** нажмите **Добавить**.
3. Укажите email, пароль и при необходимости серверы IMAP/SMTP.
4. Для самоподписанных сертификатов включите опцию игнорирования ошибок TLS.

После добавления ящик появится в списке; для него можно задать имя, цвет бейджа, подпись и отображаемое имя отправителя.

## Данные

Локальная SQLite-база создаётся в `data/mailer.db`. В ней хранятся:

- учётные записи ящиков (пароли в зашифрованном виде);
- метки и привязки меток к письмам;
- контакты.

Содержимое писем в базу не записывается.

## Структура проекта

```
src/
├── app/              # Next.js App Router, API routes
│   └── api/          # REST API (accounts, emails, folders, labels, contacts)
├── components/       # UI: Sidebar, EmailList, EmailViewer, ComposeModal, SettingsModal
└── lib/              # БД, IMAP, SMTP, шифрование, типы
data/                 # SQLite (создаётся автоматически)
```
