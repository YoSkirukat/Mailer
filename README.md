# Mailer

Единый веб-клиент для работы с несколькими почтовыми ящиками через IMAP и SMTP. Письма не хранятся в базе — загружаются по запросу с сервера.

## Возможности

### Ящики и папки

- **Несколько ящиков** — Yandex, Mail.ru, Gmail, Outlook или произвольный IMAP/SMTP
- **Папки** — входящие, отправленные, архив, корзина, спам
- **Счётчики непрочитанных** — у папок и ящиков в боковой панели
- **Автообновление** — список писем обновляется каждые 20 секунд (тихо, без мигания интерфейса)

### Письма

- **Просмотр** — HTML и текст, вложения, предпросмотр PDF и изображений в отдельном окне
- **Действия** — ответить, переслать, удалить, в архив, в спам / не спам
- **Массовые операции** — выделение нескольких писем чекбоксами, групповое удаление, архивирование, спам
- **Очистить спам** — удаление всех писем в папке «Спам»
- **Фильтр списка** — все письма или только непрочитанные
- **Поиск** — по всем ящикам и папкам (кроме спама), с бейджами папок в результатах

### Ярлыки

- Пользовательские **ярлыки** с цветом
- Назначение ярлыков письмам вручную или через контекстное меню
- Отображение ярлыков в списке писем и в окне просмотра
- Фильтрация списка по ярлыку

### Фильтрация почты

В **Настройки → Фильтрация** можно создавать правила для входящих:

- **Условия** — поле (от, кому, тема, текст), оператор (содержит / не содержит / равно), область (все правила / любое / все сообщения)
- **Действия** — переместить в папку, отправить копию на email, пометить прочитанным, удалить, установить ярлык
- Включение и выключение правил без удаления
- Фильтры срабатывают автоматически к письмам во входящих. Каждый фильтр применяется к конкретному письму **строго один раз** — повторно не срабатывает, даже если изменить правило или снять ярлык вручную.

### Составление писем

- Новое письмо, ответ, пересылка
- Вложения, подпись, отображаемое имя отправителя
- Выбор ящика-отправителя при нескольких подключённых аккаунтах

### Прочее

- **Контакты** — сохранение адресов из писем
- **Уведомления** — браузерные оповещения о новых письмах во входящих (когда вкладка неактивна)
- **Тёмная тема**
- **Настройки ящиков** — имя, цвет бейджа, подпись, имя отправителя, удаление

## Требования

- Node.js 18+
- npm

## Установка и запуск

```bash
cd Mailer

npm install

cp .env.example .env.local
# Сгенерировать ключ: openssl rand -hex 32
# Вставить его в ENCRYPTION_KEY в .env.local

npm run dev
```

Приложение откроется по адресу [http://localhost:3000](http://localhost:3000).

## Переменные окружения

| Переменная       | Описание |
|------------------|----------|
| `ENCRYPTION_KEY` | Ключ шифрования паролей ящиков (минимум 32 символа). Сгенерировать: `openssl rand -hex 32` |

## Добавление почтового ящика

1. Откройте **Настройки** в боковой панели.
2. На вкладке **Почтовые ящики** нажмите **Добавить ящик**.
3. Укажите email, пароль и при необходимости серверы IMAP/SMTP.
4. Для самоподписанных сертификатов включите опцию игнорирования ошибок TLS.

После добавления ящик появится в списке; для него можно задать имя, цвет бейджа, подпись и отображаемое имя отправителя.

## Настройка фильтров

1. **Настройки → Фильтрация → Создать фильтр**.
2. Задайте название, область применения и одно или несколько правил.
3. Добавьте действия (например, «Установить ярлык» или «Отправить копию сообщения на»).
4. Сохраните и убедитесь, что переключатель фильтра включён.

Фильтры срабатывают при загрузке входящих и при открытии письма. Пересылка копии выполняется через SMTP ящика, в который пришло письмо.

## Данные

Локальная SQLite-база создаётся в `data/mailer.db`. В ней хранятся:

- учётные записи ящиков (пароли в зашифрованном виде);
- ярлыки и привязки ярлыков к письмам;
- правила фильтрации и журнал уже обработанных писем (чтобы не применять фильтр повторно);
- контакты.

Содержимое писем в базу не записывается.

## Структура проекта

```
src/
├── app/
│   ├── api/          # REST API
│   │   ├── accounts/ # ящики
│   │   ├── emails/   # письма, вложения, массовые действия
│   │   ├── filters/  # правила фильтрации
│   │   ├── folders/  # счётчики непрочитанных
│   │   ├── labels/   # ярлыки
│   │   └── contacts/
│   ├── page.tsx      # главный интерфейс
│   └── view/         # просмотр вложений
├── components/       # Sidebar, EmailList, EmailViewer, ComposeModal,
│                     # SettingsModal, FilterSettingsPanel, …
└── lib/              # db, imap, smtp, filters-db, filter-engine, …
data/                 # SQLite (создаётся автоматически)
```
